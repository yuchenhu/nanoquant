## 二、WAP 构造：为什么 PnL 依赖衰减没有增量信息

### 2.1 简单指数衰减 WAP（业内标准做法）

```
WAP(T) = Σ [V_t × e^(-λ(T-t)) × P_t] / Σ [V_t × e^(-λ(T-t))]
```

一个参数：λ，等价于半衰期。半衰期 = 20 个交易日 → λ = ln(2)/20 ≈ 0.035。

### 2.2 PnL 依赖衰减 WAP（学术模型，业内不用）

理论思路：浮盈者卖得快（处置效应），浮亏者持有不卖。对每一天的成交量施加盈亏状态依赖的衰减。

为什么业内不用：
1. **PnL 依赖衰减只是趋势强度的缩放**。趋势上涨时，浮盈加速衰减 → WAP 被拉向近期高价 → 当前价格偏离 WAP 的幅度比简单衰减版小。趋势下跌时，浮亏减慢衰减 → WAP 保留了更多远期高价 → 价格低于 WAP 的幅度比简单版更大。简单版和 PnL 版在三种场景（涨/跌/震荡）下的**排序完全一致**，只有幅度差异——而这个幅度差异等价于趋势强度本身，不提供增量信息。
2. **处置效应参数每轮行情都变**。δ_gain（浮盈时卖出加速的强度）在牛市中变小、熊市中变大、极端行情中完全失效。用历史回测拟合的参数在新 regime 中不成立。
3. **边际增益 ≈ 0**。PnL 版对简单版的残差 IC 在 A 股实测中不足以覆盖参数估计的不确定性。

**结论**：用简单指数衰减 WAP。复杂版排序和简单版一致，没有增量信息。

### 2.3 WAP 偏离度的定义

```
wap_dist = (Close - WAP_60) / ATR_20
```

用 ATR 标准化，消除价格尺度差异。ATR 标准化后的偏离度在 ETF 之间可比较。

**物理含义**：
- `wap_dist > 0`：价格在 WAP 上方 → 平均持仓者浮盈 → 存在获利了结压力，上行需要更强的资金推动
- `wap_dist < 0`：价格在 WAP 下方 → 平均持仓者浮亏 → 存在解套盘压力或支撑（取决于趋势方向）

**阈值（starting point，需回测校准）**：
- `|wap_dist| < 1.0`：偏离小，价格在成本区附近，筹码结构健康
- `1.0 ≤ |wap_dist| < 2.0`：偏离中等，趋势在延伸，需结合 regime 判断
- `|wap_dist| ≥ 2.0`：偏离大，价格已远离市场平均成本，脆弱性上升——不管涨跌方向

为什么选 1.0/2.0：ATR 本身就是 20 日平均振幅，2 ATR 约等于正常市况下 2 个标准差的移动。价格偏离成交量加权成本超过 2 个典型日波动区间 = 统计上的异常。

### 2.4 半衰期 20 天的来源与校准

20 个交易日 ≈ 一个自然月。选取逻辑：
- **10 天**（半月）：太短，对近期噪音过度敏感，WAP 和短期均线高度相关→信息冗余
- **20 天**（一月）：覆盖一个完整交易周期，在敏感度和稳定性之间取中庸
- **40 天**（两月）：太钝，2015 年股灾级别的大波动才会让 WAP 明显移动
- **60 天**（一季）：WAP 几乎退化为长期均线，失去了"成本区"的动态含义

初始值用 20 天，回测时应纳入参数扫描（Section 七），扫 10/15/20/25/30/40 确认稳健性。

**60 天窗口的选择**：60 ÷ 20 = 3 个半衰期，覆盖约 87.5% 的指数衰减权重，足够捕捉主要成本区而不被远古数据污染。窗口增量（如 100 天→5 个半衰期）边际贡献极小。

## 三、Regime 检测：ADX + 滞回双阈值

### 3.1 为什么用 ADX

ADX 衡量趋势强度（不是方向）。ADX > 25 → 有趋势，ADX < 20 → 无趋势（震荡）。直接回答"当前适不适合做趋势跟踪"。

### 3.2 ADX 的问题是阈值附近跳动

单阈值（比如 25）二值化一个连续指标，在阈值附近的微小波动会导致 regime 标签反复切换。这不是 ADX 的问题——是二值化本身的问题。加平滑（5 日均值）只是把跳动从每天变成每两三天，治标不治本。

### 3.3 滞回双阈值（hysteresis）

```
上阈值 = 26-28  # 超过这个才确认进入趋势
下阈值 = 18-20  # 跌破这个才确认回到震荡

if ADX > 上阈值:
    regime = trending
elif ADX < 下阈值:
    regime = ranging
else:
    regime = 保持上一期的值（不改变状态）
```

中间灰色地带（20-28）不触发切换。ADX 从 29 跌到 23、涨回 27 → 状态保持 trending，一次不跳。

**用上一期的值兜底，牺牲确认速度换取标签稳定性。** 对周频 ETF 轮动，晚 1-2 天确认 regime 切换的代价可忽略。

### 3.4 阈值怎么定

把回测期的所有 ADX 值按事後已知的 regime 分成两组（趋势期 vs 震荡期）：
- 下阈值 = 震荡期 ADX 分布的 90 分位数
- 上阈值 = 趋势期 ADX 分布的 10 分位数

不拍脑袋。宽度 = 上阈值 - 下阈值，自然产生。

### 3.5 不要往 regime 定义里加成交量/宽度

成交量、宽度等指标不进 regime 定义，单独作为信号或辅助过滤维度。混在一起会导致：ADX 阈值和成交量阈值的交互不可拆解、调参空间指数爆炸。保持维度正交——regime 只管方向性（趋势还是震荡），其他维度回答"这个趋势/震荡有多可靠"。

### 3.6 Regime 的粒度：指数级别 + 映射到 ETF

在指数（宽基 + 行业）级别算 ADX，ETF 通过招募说明书里的跟踪指数做先验映射。不用 beta/R² 拟合映射——跟踪指数是发行时给定的，不需要估计。

指数算 regime 的优势：不存在 ETF 申赎对成交量/价格的污染，成分股数据（宽度、离散度）现有。

## 四、条件 IC 框架

### 4.1 条件 IC 是因子评估工具，不是策略执行工具

使用场景：研发阶段，评估一个候选因子是否值得嵌入策略。

```
for 每个 regime（趋势/震荡）:
    条件IC = corr(因子值 | regime, 未来N日收益 | regime)
```

### 4.2 四种分类

| 条件 IC 模式 | 含义 | 处理 |
|---|---|---|
| 两种 regime 下 IC 方向一致、都显著 | 不需要 regime 条件 | 任何时候都能用 |
| 一种 regime 下显著、另一种 ≈ 0 | 只在特定 regime 有效 | 限定在有效 regime 使用 |
| 两种 regime 都显著但方向相反 | 信号方向依赖 regime | 信号符号 × regime 方向 |
| 两种 regime 都不显著 | 没有预测力 | 扔掉 |

### 4.3 条件 IC 的粒度

用 ETF 自身级别做 regime 标签（单 ETF 自己算 ADX + 滞回），不做全市场统一标签。不同 ETF 之间的 regime 差异本身就是信息——如果一只在震荡、一只在趋势，轮动才有意义。

ETF 历史不足 60 天（上市太新），用跟踪指数的 ADX 代理。

## 五、因子架构：动量主引擎 × WAP/量比调制器

### 5.1 三层分工

```
动量（截面排序） —— 主引擎，决定"买谁"
    ↓ 选出方向向上的候选
WAP 偏离（时序过滤） —— 第一调制器，决定"成本距离是否安全"
    偏离大 → 不管量比如何都降权，不试图区分主升浪还是顶部
量比（时序确认） —— 第二调制器，决定"资金参与方向"
    方向由 regime 状态决定（突破日正、中途负、高位强负）
         ↓
entry_quality = f(wap_dist, vol_ratio | trend_state, z_score)
    ↓
final_weight = momentum_weight × entry_quality
```

### 5.2 为什么量比不能当主因子

量比的**全样本 IC ≈ 0**（Section 八-C 已验证：突破日 +0.15，趋势中途 -0.08，高位 -0.15，盘整 ≈ 0——各 regime 的 IC 在正负之间抵消）。作为主因子跑截面排序 = 拿噪音当信号。

量比值钱的用法是作为**条件调制器**——它的 IC 方向由 `trend_state × z_score` 的 regime 格点决定。独立使用时没有信息量，结合 regime 状态后才有。
同理，WAP 偏离的全样本 IC 也接近零——偏离大可以是主升浪的起点，也可以是顶部。WAP 偏离的价值在"偏离大时降权、偏离小时正常配置"，即**下行保护**，不是**上行预测**。

### 5.3 正确的因子角色表

| 信号 | 角色 | 用法 | 独立 IC 预期 |
|------|------|------|------------|
| 动量 | 主引擎 | 截面排名，选出 top K | 显著正（~0.05-0.10） |
| WAP 偏离度 | 下行保护 | 偏离 > 2 ATR → 降权 | ≈ 0（不预测方向） |
| 量比 | 条件调制 | 方向由 regime 决定 | ≈ 0（全样本抵消） |
| 宽度 | 质量确认 | 广度 > 50% 加分 | 微弱正 |
| 成分股离散度 | 风险信号 | 离散度飙升 → 降权 | 负（尾部） |
| 融资买入/成交比 | 情绪验证 | 高位+高融资 → 降权 | 负（尾部） |

**关键认知**：WAP 偏离和量比不是在和动量"比赛谁 IC 更高"——它们是在动量选出的候选池里做减法。评价标准不是排序 IC，是"加了这个调制器后，动量组合的回撤有没有降低"（RAS_bear）。

### 5.4 当前已验证的信号池（需要条件 IC 逐个确认）

| 信号 | 类型 | 构造 |
|------|------|------|
| 动量 | 截面排序 | N 日收益率排名 |
| WAP 偏离度 | 时序过滤 | (Close - WAP_60) / ATR_20 |
| 量比 | 时序确认 | 5 日日均成交量 / 60 日日均成交量 |
| 宽度 | 时序加分 | 指数内上涨家数 / 总家数 |
| 成分股离散度 | 时序 | 成分股收益率标准差 |
| 融资买入/成交比 | 时序 | （来自 [[融资融券与两融数据]]） |
| 期货基差偏离度 | 时序 | （来自 [[股指期货基差与期现联动]]） |

每个候选信号先过条件 IC 这一关。不过关的不进入任何 regime 的因子池。

## 六、WAP × 量比的 regime 调制矩阵（替代旧四象限） ^deprecated-4q

> ⚠️ **2026-07-20 重写**：旧版四象限（缩量温和涨/放量突破等，仅按量比×WAP做2×2分格）已被本节取代。旧版未区分 regime，导致趋势中途放量被标为"突破确认"（仓位1.0），与 Section 八-C 的条件 IC（趋势中途放量 IC=-0.08）矛盾。新版按 `trend_state × Z_score × wap_dist` 三个维度联合定义。

### 6.1 WAP 偏离的第一层调制：下行保护（与量比无关）

WAP 偏离大的 ETF 直接降权，**不等到看量比**。这是纯下行保护——价格已远离市场平均成本，不管后面是主升浪还是顶部，脆弱性已经上升。

```
if |wap_dist| ≥ 2.0:
    wap_penalty = 0.5         # 偏离极大 → 仓位打五折
elif |wap_dist| ≥ 1.0:
    wap_penalty = 0.8         # 偏离中等 → 仓位打八折
else:
    wap_penalty = 1.0         # 在成本区附近 → 不折
```

### 6.2 量比的第二层调制：方向由 regime 决定

在 WAP 偏离不过分的前提下（wap_penalty > 0），量比的 IC 方向由 `trend_state` 和 `Z_score` 共同决定。此处的 regime 定义来自 Section 八-C。

```
vr = volume_ratio = V_5d_avg / V_60d_avg
vr_high = vr > 1.5   # 放量阈值（starting point，需回测校准）
vr_low  = vr < 0.7   # 缩量阈值（同上）
```

**1.5 和 0.7 的来源**：历史 A 股日成交量的 70 分位约 ~1.3 倍均量、30 分位约 ~0.8 倍均量。向上取 1.5（保守）、向下取 0.7（保守），留出中间灰色地带。对标八-C 中突破确认的 1.5 阈值。

### 6.3 联合调制矩阵

```
trend_state=False（盘整/震荡）
  ├── 突破日（刚翻 True）：
  │     vr > 1.5 → vol_factor = 1.5  （唯一正信号，放量突破确认）
  │     vr ≤ 1.5 → vol_factor = 1.0  （无放量不加分，不扣分）
  └── 非突破：
        → 不参与（ADX 未确认趋势，趋势策略无 edge）

trend_state=True（趋势中）
  ├── Z_score < 1.5（趋势中途）：
  │     vr > 1.5 → vol_factor = 0.8   （放量=散户追入，降权 20%）
  │     vr < 0.7 → vol_factor = 1.0   （缩量健康，正常配置）
  │     0.7-1.5 → vol_factor = 1.0    （正常量，不调）
  └── Z_score ≥ 1.5（高位延伸）：
        vr > 1.5 → vol_factor = 0.5   （放量=大概率派发，降权 50%）
        vr < 0.7 → vol_factor = 0.7   （高位缩量也脆弱，降权 30%）
        0.7-1.5 → vol_factor = 0.8    （正常量但高位，降权 20%）
```

### 6.4 entry_quality 合成公式

```
entry_quality = wap_penalty × vol_factor

# 最终落入 0.7 × 0.8 = 0.56 到 1.0 × 1.5 = 1.5 之间
# 为什么是乘法：WAP 偏离大 + 高位放量 = 两份独立的风险叠加，应该叠加处罚
# 为什么不是 min：突破日 wap_penalty=1.0 且 vol_factor=1.5 → min=1.0，丢掉了量的正信号
# 为什么不是求和：和会模糊"偏离大+放量"的双重风险

entry_quality = clamp(entry_quality, 0.3, 1.5)
# 封底 0.3（不归零——动量已确认方向正确，不因筹码过滤完全否定）
# 封顶 1.5（突破放量可以加 50%，但不加更多——单因子加成有上限）

final_weight = momentum_rank_weight × entry_quality
```

### 6.5 验证路径

这套调制器的评价不走排序 IC。验证分两步：

1. **RAS_bear**：加了调制器后，动量组合在熊市月的平均收益是否比纯动量组合更高（即少亏了多少）
2. **条件覆盖测试**：按 `|wap_dist|` 分桶，看各桶的 forward return 分布——桶 ≥2.0 的右尾是否明显更肥（证明降权确实避开了暴跌月份）

## 七、三个优化维度（非串行）

| 维度 | 回答的问题 | 优化对象 |
|------|-----------|---------|
| Regime 定义 | 当前是什么市场状态？ | ADX 上/下阈值、滞回宽度、是否需要波动率聚集辅助 |
| 因子构造 | 每个因子怎么从原始数据算？ | WAP 半衰期、量比窗口比、宽度分母 |
| 因子组合 | 多个有效信号怎么集成？ | 等权/IC 加权/动态加权、因子间相关性惩罚 |

三个维度有交互效应——regime 定得好可能让原来不显著的因子变显著；因子构造改参数可能改变 regime 切换频率。不要按"先 regime、再因子、再组合"的顺序孤立优化。回测时三个维度一起扫参数空间。

## 八、嵌入 ETF 轮动的完整流程

### 因子研发阶段（一次性）

1. 对所有候选信号跑条件 IC（按 ETF 自身 regime 分组）
2. 分类：regime-independent / regime-dependent / 垃圾
3. 确定每个信号在哪个 regime 下有效、方向是什么

### 策略执行阶段（每次调仓）

```
Step 0 — Regime 判断：
  每只 ETF 计算 ADX，应用滞回双阈值 → 打标签 trending/ranging
  计算 Z_score

Step 1 — 截面排序：
  在 trending regime 的 ETF 中，按动量得分排序 → 选出前 K 只候选

Step 2 — 候选池准入：
  Z_score > 0（价在 MA50 上方，不做左侧交易）

Step 3 — entry_quality 调制：
  wap_penalty = f(|wap_dist|)           # Section 6.1（或增强版 八-C2）
  vol_factor  = g(vol_ratio | trend_state, Z_score)  # Section 6.3
  entry_quality = clamp(wap_penalty × vol_factor, 0.3, 1.5)  # Section 6.4

Step 4 — 仓位分配：
  final_weight = momentum_rank_weight × entry_quality
  → ATR 标准化：pos = risk_budget × final_weight / ATR_20

Step 5 — 无候选时的 fallback：
  如果 trending regime ETF 数量 < 最低持仓数 → 缩减总仓位或转货币 ETF
```

## 八-A、ADX滞回状态 vs 原始ADX值：各司其职

### 第一性原理

Regime 的定义需要一个**稳定的二值标签**。原始 ADX 是连续值，在阈值附近日常波动——今天 21、明天 19，如果 regime 跟着原始值跳，条件 IC 的计算底座就塌了。滞回给出的是一个不跳动的二值状态；原始 ADX 的值在别的地方有用（趋势强度微调）。

两者不冲突，用途不同：

| | 滞回状态 `trend_state` | ADX 原始值 `adx_raw` |
|---|---|---|
| 类型 | 二值（True/False） | 连续值 |
| 用途 | 定义 regime，决定量比 IC 方向 | 趋势强度连续微调 |
| 在哪用 | 突破事件捕获、条件 IC 分组 | 仓位系数缩放 |
| 例子 | `trend_state=True` → 趋势中途或高位 | `adx_raw=25` → 标准仓位；`adx_raw=45` → 可能过热 |

### 量化表达

```
# 滞回状态递推
if adx_raw > adx_upper:      # 上阈值，如 22
    trend_state = True
elif adx_raw < adx_lower:    # 下阈值，如 18
    trend_state = False
else:
    trend_state = trend_state_prev   # 灰色地带，保持上一期

# 突破日捕获：状态从 False 跳变到 True 的那一天
breakout = (trend_state_prev == False) and (trend_state == True)
breakout_confirmed = breakout and (volume_ratio > 1.5)
```

突破日只会被识别一次——状态变迁的那一刻。用原始 ADX 上穿 20 会导致同一周内被识别出三次"突破"，无意义。

## 八-B、Z-score：趋势位置的量化

### 第一性原理

ADX 回答"有没有趋势"，Z-score 回答"趋势走了多远"。两个维度正交——你可以在强趋势里价格刚启动（ADX=30, Z=0.3），也可以在弱趋势里价格已经拉得很远（ADX=21, Z=2.1）。

用 Z-score 而非 RSI 的原因：RSI 是隐式平滑，参数敏感、上下边界非线性。Z-score 含义直接——价格离均线几个标准差。1.5σ = 有点远，2σ = 极端。

### 量化表达

```
Z_score = (Close - MA_50) / StdDev_50

# MA_50 = 过去50个交易日收盘价均值
# StdDev_50 = 过去50个交易日收盘价标准差
```

**关键**：每支 ETF 用自身历史数据滚动计算 Z-score，不做跨 ETF 统一阈值。科创 50 的日波动天然是沪深 300 的 1.5-2 倍——统一阈值会把小盘 ETF 永远标成"高位"。

### Z-score 两层用途

```
# 1. 准入条件（硬性门槛）
can_enter = Z_score > 0   # 价格在 MA50 上方，不做左侧交易

# 2. 位置分档（连续微调）
if Z_score < 1.5:
    regime_position = "mid_trend"  # 趋势中途
else:
    regime_position = "extended"   # 高位延伸
```

Z_score > 0 是准入，不是排序因子——它不管 0.2 和 0.8 的精细区分，只管"能不能进候选池"。机制：Z_score ≤ 0 的 ETF 处于下跌或弱势横盘，买入的唯一理由是赌反转 → 反转概率远低于趋势延续 → 负 EV。

---

## 八-C、量比的条件 IC：Regime 翻转

### 第一性原理

A 股散户主导边际定价权。放量的含义完全取决于当前 regime：

- **盘整后突破**：放量 = 资金表态 → 正信号（Livermore 关键点）
- **趋势中途**：放量 = 散户追入 → 负信号（放量即高潮）
- **高位延伸**：放量 = 派发/出货 → 强负信号

同一个量比信号，IC 从 +0.15 到 -0.15 完全翻转。不按 regime 分层看量价关系 = 拿正负 IC 互相抵消后的噪音当信号。

### 量化表达

```
volume_ratio = V_5d_avg / V_60d_avg

# 条件 IC = corr(volume_ratio | regime, return_t+1 | regime)
```

### 四象限呈现

两个轴：ADX 滞回状态（高/低） × Z-score（高/低）

```
              ADX低 (<20)                  ADX高 (>=20)，trend_state=True
           ┌──────────────────────┬──────────────────────────┐
Z_score低  │  盘整筑底               │  趋势中途                  │
 (<1.5)    │  量比IC: ≈ 0           │  量比IC: -0.08             │
           │  操作: 等待             │  操作: 持仓不动，放量降权    │
           ├──────────────────────┼──────────────────────────┤
Z_score高  │  高位震荡               │  高位加速赶顶                │
 (>=1.5)   │  量比IC: ≈ 0           │  量比IC: -0.15             │
           │  操作: 不参与           │  操作: 减仓或跳过            │
           └──────────────────────┴──────────────────────────┘

突破事件（箭头）：trend_state 从 False→True 跳变日
  if breakout and volume_ratio > 1.5:
      量比IC ≈ +0.15（最干净的买点）
```

### 各 regime 的量比 IC 数值

| Regime | 触发条件 | 量比→次日收益 IC | 方向 | 操作 |
|--------|---------|-----------------|------|------|
| 盘整 | `trend_state=False` | ~ 0 | 弱正/无 | 不参与，除非刚好突变为突破日 |
| 突破 | `trend_state` 刚翻 True + `volume_ratio > 1.5` | +0.10 ~ +0.20 | 强正 | 全仓跟进 |
| 趋势中途 | `trend_state=True`, `Z_score < 1.5` | -0.05 ~ -0.10 | 负 | 缩量涨好，放量涨降权 20% |
| 高位延伸 | `trend_state=True`, `Z_score >= 1.5` | -0.10 ~ -0.20 | 强负 | 降权 50% 或跳过 |

### 注意：条件 IC 不是一次性算完就刻舟求剑

每个月回滚一次条件 IC，确认四个 regime 的 IC 方向和相对大小仍然成立。某个 regime 的 IC 从负翻正 → 市场微观结构在变 → regime 定义需要重新校准。

## 八-C2、WAP 偏离度 × Z-score：成本距离的阶段含义

### 第一性原理

同一个 WAP 偏离度，在趋势的不同阶段含义不同：

- **刚突破时偏离 1.5 ATR**：突破力度强，资金在积极抢筹 → 偏积极
- **高位时偏离 1.5 ATR**：价格已经跑了很远，获利盘堆积 → 偏危险

WAP 偏离度和 Z-score 之间存在交互效应——Section 六的 `wap_penalty`（仅按 `|wap_dist|` 绝对值打折）是简化版，未利用 Z-score 的阶段信息。此处提供一个**可选的增强版本**（需回测验证边际增益是否足以覆盖复杂度）。

### 量化表达

```
# 增强版 wap_penalty（替代 Section 6.1 的简化版）
if Z_score < 1.5（趋势中途）:
    if |wap_dist| ≥ 2.0:  wap_penalty = 0.6   # 中途已拉远 → 偏危险
    elif |wap_dist| ≥ 1.0: wap_penalty = 0.9   # 中途温和偏离 → 略降
    else:                  wap_penalty = 1.0
elif Z_score ≥ 1.5（高位延伸）:
    if |wap_dist| ≥ 2.0:  wap_penalty = 0.4   # 高位+拉远 → 强降权
    elif |wap_dist| ≥ 1.0: wap_penalty = 0.7   # 高位+中等偏离 → 降权
    else:                  wap_penalty = 0.85  # 高位但成本区窄 → 微降
```

**简化版 vs 增强版的选择**：Section 6.1 的简化版（不看 Z-score）和此处的增强版的排序结果可能高度一致。建议先用简化版跑回测，看简化版的 RAS_bear 是否已经够好，再决定是否加 Z-score 维度。不要为了"更有道理"而增加没增量信息的参数。

### 联合调制矩阵的完整形式

自下往上：动量选出 top K → `wap_penalty(Z_score)` 打折 → `vol_factor(trend_state, Z_score, volume_ratio)` 二次调制 → `entry_quality = wap_penalty × vol_factor`。

两个调制器完全并行处理——WAP 管成本距离、量比管资金方向，互不交叉，只在最后的乘法汇合。

## 八-D、两层 Regime 架构：全A 层 vs 单支 ETF 层

### 第一性原理

全A 放量涨和单支 ETF 放量涨的机制完全不同，不能共用一套 regime 框架：

- **全A 放量涨**：增量资金（北向、两融、新开户）涌入，蛋糕变大了 → 中期正信号
- **单支 ETF 放量涨**：存量资金从别的 ETF 搬过来，蛋糕没变大 → 趋势中途的负信号

因此仓位管理和品种选择必须分层，两层各用各的 regime。

### 量化表达

```
# Layer 1: 全A → 决定总仓位中枢
total_position_cap = f(全A_trend_state, 全A_Z_score)

  全A_trend_state=False         → max_position = 30%
  全A_trend_state=True, Z<1.5   → max_position = 70%
  全A_trend_state=True, Z>=1.5  → max_position = 30%

# Layer 2: 单支 ETF → 决定品种权重
etf_weight_raw = momentum_rank_weight × entry_quality(ETF_trend_state, ETF_Z_score, volume_ratio)

# 最终权重 = 两层合并
final_weight = total_position_cap × normalize(etf_weight_raw)
```

### 两层职责分明

```
全A regime 管：总仓位上限、是否可以空仓
ETF regime 管：选谁、买多少、什么时候降权/退出
```

两层各用各的 ADX + 滞回 + Z-score。全A 在盘整时不等于不能交易——某些 ETF 自身在趋势中途，可以轻仓。全A 在高位延伸时必须压总仓位，即便个别 ETF 看起来还在健康上涨——系统性风险会拖累所有品种。

## 九-A、五维度面板的定位：宏观监控 ≠ 策略执行

### 为什么 36 列和 2 个指标不冲突

五维度面板（价量波估值资金，36 列，月频）和 ADX+Z-score（2 指标，日/周频）是不同决策层级的工具：

```
五维度面板（月频）→ 战略层：
  - 总仓位中枢的月度review
  - 风格偏向（大盘vs小盘）判断
  - 极端估值预警（PE 5y分位 > 90%）
  - 资金面全貌（北向、两融趋势）
  - 叙事验证（与识别主线体系对接）

ADX + Z-score（日/周频）→ 执行层：
  - ETF 轮动的 regime 条件 IC
  - 突破信号捕获
  - 量比方向判断
  - 权重调整
```

### 为什么执行层不用多维度合成

多维度合成为一个 regime 标签存在递归问题：五个维度的权重本身依赖于"当前是什么市场状态"——你在用一个依赖答案的问题去求解答案。一个具体例子：

```
2019年Q1，五维度：
  价：涨（看多）
  量：放大（趋势中途 = 看空）
  估值：历史低位（看多）
  资金：北向大额流入（看多）
  波：升高（看空）

合成结果：3看多 vs 2看空 → "总体上偏多"
但量维度在趋势中途是负IC → 你的"看多"里已经混入了反向因子
```

ADX 不需要权重——它有明确的滞回二值输出，不会给"总体上偏多但有几个维度不太对劲"的模糊信号。

### 五维度面板的价值：不是被替代，是归位

五维度面板回答的是"水位"问题——钱在进还是在出，而不是"买哪个"。月度 review 时打开面板看大图，日频策略执行时用 ADX+Z-score——两个层级正确分工，不互相替代。

## 九-B、最终完整嵌入流程（2026-07-20 修正版）

```
Layer 0 — 宏观水位（月频，五维度面板 + market_regime_roadmap）：
  全A PE分位、北向趋势、两融余额 → 决定总仓位中枢区间
  PB≥95% | PE≥95% | Gini≥0.68 → 一票空仓
  叙事拥挤度（周频）→ narrative_multiplier [0.6-1.0]

Step 0 — 全A Regime（日频，ADX+滞回）：
  全A trend_state, Z_score → 决定当日仓位上限

Step 1 — ETF Regime（日频，每支 ETF 独立计算）：
  ADX 滞回 → trend_state, Z_score → 打标签

Step 2 — 截面排序 + 准入：
  Z_score > 0 & trend_state=True 的 ETF → 动量排名 → 前 K 支

Step 3 — entry_quality 调制（Section 六，替代旧版"时序过滤"）：
  wap_penalty = f(|wap_dist|, Z_score)    # Section 6.1 或增强版 八-C2
  vol_factor  = g(vol_ratio | trend_state, Z_score)  # Section 6.3
  entry_quality = clamp(wap_penalty × vol_factor, 0.3, 1.5)

Step 4 — 两层权重合并：
  final_weight = total_position_cap × etf_rank_weight × entry_quality
  然后用 ATR 标准化：pos = risk_budget × final_weight / ATR_20
```
