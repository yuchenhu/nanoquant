# ETF轮动 Alpha 因子库

> 创建：2026-07-14 ｜ 目标：宽基+行业指数池的OHLCV因子候选集

---

## 一、第一性原理层

**ETF轮动alpha的本质是时序上识别相对强势的指数。** 与股票截面因子不同，这里的每个观测点是一条指数的时间序列，截面上的"股票数"远小于股票池（几十 vs 几千），但时序长度相同。

这决定了两个设计约束：

1. **样本量不在截面堆积，在时序上深耕**——每个因子必须从OHLCV里榨出比「过去N日收益率」更多的信息
2. **因子间信息源必须独立**——池子小，如果两个因子高度相关，多因子合成的收益基本为零（over-diversification在几十个标的上不成立）

## 二、因子目录

### 因子1：趋势一致性动量（Momentum × R²）

**来源**：[[ETF动量质量因子]]

**因果链**：纯动量只度量「涨了多少」，不关心「怎么涨的」。一条陡拉涨停的路径和一条稳步攀升的路径，在纯动量下得分相同。前者后续反转概率高，后者趋势持续性高。R² 区分了这两种路径。

**公式**：

```
ret_Nd = ln(P_t / P_{t-N})          // N日对数收益率（N=20或40）

// 对过去N日收盘价做OLS线性回归
P_i = α + β × i + ε_i,  i = 1..N

R² = 1 - Σε² / Σ(P_i - P̄)²     // 回归拟合优度

score = sign(ret_Nd) × |ret_Nd| × R²
```

**IC预期**：正向，IC ≈ 0.05-0.08，比单窗口纯动量高约30%

**证伪条件**：R²在A股宽基指数上的区分度可能弱于个股（指数本身是数百只股票的均值，路径天然更平滑）

**数据需求**：OHLCV 中的 C（收盘价），Tushare `index_daily`

### 因子2：动量期限结构（Term Structure of Momentum）

**因果链**：不是判断"有没有趋势"，而是判断"趋势在加速还是减速"。加速上涨 → 趋势健康；上涨但减速 → 动能衰竭，即将反转。

**公式**：

```
ret_5d  = ln(P_t / P_{t-5})
ret_10d = ln(P_t / P_{t-10})
ret_20d = ln(P_t / P_{t-20})
ret_40d = ln(P_t / P_{t-40})
ret_60d = ln(P_t / P_{t-60})

tsmom = (ret_5d  - ret_20d)      // 短期超额 = 加速项
      + (ret_10d - ret_40d)      // 中期超额 = 持续项

// 方向确认项（可选，防反转误判）
trend_ok = (ret_60d > 0) ? 1 : 0.5
           // 长期仍在涨 → 满权重；长期在跌但短期在涨 → 半权重（可能是反弹）

score = tsmom × trend_ok
```

**四象限**：

```
               ret_60d > 0                  ret_60d < 0
         ┌──────────────────────┬────────────────────────┐
tsmom>0  │ 加速牛市（最优）        │ 反弹初期（次优）          │
         │ IC ≈ +0.08-0.12       │ IC ≈ +0.03-0.05         │
         ├──────────────────────┼────────────────────────┤
tsmom<0  │ 趋势生锈（警告）        │ 全面下跌（回避）          │
         │ IC ≈ 0 或略微负        │ IC ≈ -0.03-0.00         │
         │ 趋势可能在转，降低仓位   │ 不在多头候选池里         │
         └──────────────────────┴────────────────────────┘
```

**IC预期**：tsmom>0条件下正向IC约 0.06-0.10

**证伪条件**：在震荡市中 tsmom 频繁变号 → 假信号率高

**数据需求**：OHLCV 中的 C，Tushare `index_daily`

### 因子3：WAP偏离度 × 量比（Regime条件）

**来源**：[[筹码分布量化落地与Regime框架]]，从个股搬到指数

**因果链**：WAP是过去N日所有交易者的平均成本。价格远高于WAP = 大多数持仓者浮盈 → 抛压小、趋势可延续。但放量加速时要分regime——趋势初期放量=确认，趋势末期放量=出货。

**公式**：

```
WAP = Σ(close_i × volume_i) / Σ(volume_i)    for i in window N=20

WAP_deviation = (close - WAP) / WAP

volume_ratio = volume_today / MA(volume, 20)

量比衰减函数 g(vol_ratio, regime)：
  若 regime = "趋势初期"（ADX上升且ADX<30）：g = min(vol_ratio, 2.0)
  若 regime = "趋势中期"（ADX>30且ADX not rising）：g = max(1.0, min(vol_ratio, 1.5))
  若 regime = "趋势末期"（ADX>30且ADX falling）：g = max(0.5, min(vol_ratio, 1.0))
  // 末期放量不再加分，防止追在出货位置

score = WAP_deviation × g(volume_ratio, regime)
```

**四象限（仅限趋势初期regime）**：

```
               WAP_deviation > 0            WAP_deviation < 0
         ┌──────────────────────┬────────────────────────┐
放量     │ 浮盈+放量确认（最优）    │ 浮亏+放量换手               │
vol_hi   │ IC ≈ +0.06-0.10       │ → 可能是底部吸筹 or 恐慌出货 │
         │                      │   需结合其他因子判断          │
         ├──────────────────────┼────────────────────────┤
缩量     │ 浮盈+缩量观望            │ 浮亏+缩量阴跌               │
vol_lo   │ IC ≈ +0.01-0.03       │ IC ≈ -0.02-0.05            │
         └──────────────────────┴────────────────────────┘
```

**IC预期**：regime条件正向 IC 约 0.04-0.07

**证伪条件**：指数的WAP偏离度区分度弱于个股（指数成分股众多，均价锚的意义不如个股清晰）

**数据需求**：OHLCV 中的 C + V，Tushare `index_daily`

### 因子4：相对强度 Alpha（Residual Momentum）

**因果链**：剥离大盘影响后的超额收益才是真alpha。一个指数涨了10%，但大盘同期涨了12% → 它实际上是跑输的，不应入选。

**公式**：

```
// Step 1: 每只指数对大盘做60日滚动回归
r_i,t = α_i + β_i × r_mkt,t + ε_i,t                         t ∈ [t-59, t]

// Step 2: 计算过去N日累计残差（超额收益）
cum_excess_i = Σ ε_i,s                                           s ∈ [t-N+1, t]

// Step 3: 横截面z-score标准化
z_residual_i = (cum_excess_i - mean(cum_excess_j, j=1..K))
             / std(cum_excess_j, j=1..K)

score = z_residual_i
```

**为什么优于纯动量**：纯动量选出来的可能是被大盘带起来的（β>1），而不是自己强。残差动量剔除了"水涨船高"的成分。

**IC预期**：正向，IC ≈ 0.04-0.06

**证伪条件**：回归窗口（60日）内的β不稳定 → 残差本身含噪声

**数据需求**：指数日收益率 + 大盘日收益率（沪深300/中证全指），Tushare `index_daily`

### 因子5：成分股一致性（Constituent Agreement）

**因果链**：指数涨了≠健康。如果是仅3-5只权重股拉着涨、其他成分股在跌 → 上涨脆弱，随时可能崩塌。真正的健康上涨需要广泛的成分股参与。

**公式**：

```
// 过去N日（N=20）

total = 成分股总数

up_count     = count(成分股 20日收益 > 0)
oufperf_count = count(成分股 20日收益 > 同期大盘收益)

breadth     = up_count / total                // 上涨广度
oufperf_pct = outperf_count / total           // 跑赢大盘比例

// 注意：权重加权 vs 等权
// 市值加权指数中，即使广度<50%，只要权重股在涨，指数也可能涨
// 等权 breadth 是更好的健康度指标

agreement_score = index_ret_20d × breadth × oufperf_pct

// 如果 index_ret_20d > 0 但 agreement_score 低：
// → 小部分权重股在拉指数 → 脆弱上涨 → 后续反转概率高
```

**四象限**：

```
               breadth_高（>60%）            breadth_低（<40%）
         ┌──────────────────────┬────────────────────────┐
指数涨   │ 健康牛市（最优）         │ 虚假繁荣（警告）           │
         │ IC ≈ +0.06-0.10       │ IC ≈ -0.02~+0.02         │
         │                      │ → 权重股独拉，随时可崩      │
         ├──────────────────────┼────────────────────────┤
指数跌   │ 少数恐慌（错杀？）       │ 全面下跌（回避）           │
         │ IC ≈ 0~+0.03          │ IC ≈ -0.03-0.00           │
         │ → 可能是权重股错杀      │                           │
         └──────────────────────┴────────────────────────┘
```

**IC预期**：指数涨×高breadth条件下正向IC约 0.06-0.10

**证伪条件**：成分股数据获取和维护成本高，过期成分股列表导致信号失真

**数据需求**：成分股日线数据（`index_member_all` + `index_daily` + 个股 `daily`），Tushare

### 因子6：行业轮动速度（Sector Rotation Speed）

**因果链**：这不是单个指数的因子，而是跨指数池的横截面动态因子。当行业热点快速切换时，追趋势 = 追到一个已经被抛弃的东西。轮动速度高 → 动量因子的可靠性下降 → 仓位应降低。

**公式**：

```
// Step 1: 今天所有指数按20日收益率排序
rank_i,t = argsort(ret_20d_j,t, j=1..K)    // 排第一的是最强的

// Step 2: 计算排名变化剧烈程度
rotation_speed_t = mean(|rank_i,t - rank_i,t-5|) / K
                  // K = 池中指数总数，归一化到[0, 1]
                  // 所有行业的排名在5天内的平均绝对变化

// Step 3: 作为动量得分的折扣因子
momentum_discount = max(0.3, 1 - rotation_speed_t)

adjusted_momentum_score_i = raw_momentum_score_i × momentum_discount
```

**物理意义**：
- rotation_speed ≈ 0 → 排名稳定，趋势可靠 → 动量得分全额使用
- rotation_speed ≈ 0.5+ → 每天换龙头 → 动量得分打三折

**IC预期**：它本身不产生alpha，是alpha因子的可靠性调节器。加入后改善策略Sharpe约 0.1-0.2

**证伪条件**：rotation_speed 在A股的历史中枢可能天然偏高 → 阈值需要相对历史分位数设定，不能用绝对值

**数据需求**：全池子指数的日收益率，Tushare `index_daily`

### 因子7：隔夜 vs 日内收益分解

**因果链**：A股的隔夜收益主要反映机构定价（政策消息+海外市场+预期修正），日内收益主要反映散户博弈。隔夜动量对未来的预测力 > 全日动量。最极端的信号：隔夜涨+日内跌 = 机构在买、散户在卖 → 强买入。

**公式**：

```
overnight_ret_t = (open_t - close_{t-1}) / close_{t-1}
intraday_ret_t  = (close_t - open_t) / open_t

// 隔夜动量
on_mom = Σ overnight_ret_s        s ∈ [t-19, t]

// 日内反转（日内跌得越狠，反转越强——散户情绪）
id_mom  = Σ intraday_ret_s        s ∈ [t-19, t]

// 信号构造
if on_mom > 0 and id_mom < 0:
    score = |on_mom| + |id_mom|              // 机构买+散户卖 = 最强
elif on_mom > 0 and id_mom > 0:
    score = on_mom × 0.5                     // 机构散户一起买 = 偏强但有泡沫风险
elif on_mom < 0 and id_mom > 0:
    score = -|on_mom| × 0.5                  // 机构卖+散户买 = 危险
else:
    score = -|on_mom| - |id_mom|             // 都在卖 = 最差
```

**四象限**：

```
               on_mom > 0                  on_mom < 0
         ┌──────────────────────┬────────────────────────┐
id_mom>0 │ 合力上涨                 │ 散户接盘（危险）          │
         │ IC ≈ +0.03-0.05       │ IC ≈ -0.05-0.08          │
         │                      │ → 机构在出货给散户        │
         ├──────────────────────┼────────────────────────┤
id_mom<0 │ 机构偷偷买（最优）       │ 合力下跌（回避）           │
         │ IC ≈ +0.06-0.10       │ IC ≈ -0.03-0.05          │
         │ → 散户恐慌+机构低吸    │                           │
         └──────────────────────┴────────────────────────┘
```

**IC预期**：on_mom>0×id_mom<0 象限正向IC约 0.06-0.10

**证伪条件**：T+1制度下日内收益的信息在第二天才能行动 → 信号有1日滞后，部分alpha被吃掉

**数据需求**：OHLCV 中的 O + C，Tushare `index_daily`（需确认 `index_daily` 是否提供 `open`，如不提供则用对应 ETF 的 `open` 做 proxy）

### 因子8：非对称 Beta 五信号

**因果链**：市场对同一板块的涨跌定价是不对称的。下行beta突然跳升 → 市场正在重新定价该板块的下行风险 → 资金正在撤出定价。这不是历史描述，是定价行为的实时shift。

#### 8A：下行 Beta 变动（核心信号）

```
// 60日滚动非对称回归
r_i,t = α + β_up × max(r_mkt,t, 0) + β_down × |min(r_mkt,t, 0)| + ε_t
        // 注意：β_down 取正数，大盘跌1% → 指数跌 β_down%

β_down_change = β_down_today - β_down_20days_ago

score = -β_down_change     // 下行敏感度上升 = 减分
```

**阈值**：
```
β_down_change > 0.3  → 下行风险在快速恶化 → 强负面
β_down_change > 0.15 → 关注
β_down_change < 0    → 下行敏感度在下降 → 正面
```

**IC预期**：负向，IC ≈ -0.03 ~ -0.06（注意：是负相关，下行beta升高 → 未来收益降低）

#### 8B：非对称比率

```
beta_asymmetry = β_up / β_down

beta_asymmetry > 1.2 → 涨时跟涨、跌时抗跌 → 正面
beta_asymmetry < 0.85 → 跌时比涨时更敏感 → 负面

score = beta_asymmetry - 1.0
```

#### 8C：β_down × R² 联合信号

```
// R² 来自 8A 中同一回归的拟合优度
// 度量指数有多"看大盘脸色"

if β_down > 1.5 and R² < 0.3:        // 跌时跟跌但关系不稳定 = 局部流动性问题
    score -= 1.0                       // 不跌则已，一跌崩盘
elif β_down > 1.5 and R² > 0.7:      // 系统性脆弱，但可预期
    score -= 0.3
elif β_down < 0.8 and R² > 0.7:      // 防御型，稳健
    score += 0.3
```

**β_down × R² 四象限**：

```
               R²_高（>0.7）              R²_低（<0.3）
         ┌──────────────────────┬────────────────────────┐
β_D高    │ 系统性脆弱（中性偏负）    │ 流动性陷阱（强负面）        │
(>1.5)   │ 大盘跌它必跟跌           │ 平时不跟跌，跟的时候暴跌     │
         │                      │ → 市场不敢给它定价         │
         ├──────────────────────┼────────────────────────┤
β_D低    │ 防御型（正面）           │ 独立行情（中性）           │
(<0.8)   │ 稳定地抗跌              │ 基本不看大盘脸色           │
         └──────────────────────┴────────────────────────┘
```

#### 8D：下行捕获率（DCR）

```
// rolling 60日窗口中，仅在大盘下跌日（r_mkt < 0）上计算

大盘下跌日集合： D = {t | r_mkt,t < 0, t ∈ [today-59, today]}

sum_index_down = Σ r_i,t                              t ∈ D
sum_mkt_down   = Σ |r_mkt,t|                          t ∈ D

DCR = |sum_index_down| / sum_mkt_down

DCR > 1.2 → 下行捕获率过高 → 负面
DCR < 0.8 → 下行抗跌 → 正面
```

#### 8E：捕获率差异

```
// 同理计算上行捕获率（大盘上涨日）
UCR = sum_index_up / sum_mkt_up

capture_ratio = UCR / DCR

capture_ratio > 1.2 → 涨时比跌时弹性好 → 正面
capture_ratio < 0.8 → 跌时比涨时弹性差 → 持续跑输

// 变动信号
capture_ratio_change = capture_ratio_today - capture_ratio_20days_ago
if capture_ratio_change < -0.3:
    → 结构在恶化 → 强负面
```

**IC预期汇总**：

| 子信号 | IC方向 | 预期范围 |
|--------|--------|---------|
| β_down_change | 负 | -0.03 ~ -0.06 |
| beta_asymmetry | 正 | +0.02 ~ +0.04 |
| β×R²联合 | 条件 | 仅在R²<0.3时有效 |
| DCR | 负 | -0.03 ~ -0.05 |
| capture_ratio_change | 正 | +0.02 ~ +0.04 |

**关键警告**：A股行业的上下行beta不对称性可能不如美股显著。原因：
- 行业轮动受政策驱动远大于市场自发定价
- 涨跌停限制使下行beta的计算本身失真
- 某些行业的beta实际上反映的是「机构有没有在出货」

**优先级**：这组因子放在第6-8位——不如动量/WAP/隔夜分解硬。加入后边际改善Sharpe则保留，无增量则砍。

**数据需求**：指数日收益率 + 大盘日收益率，Tushare `index_daily`

## 四、综合打分聚合

```
// 每个因子先做横截面z-score标准化
z_j,i = (factor_j,i - mean(factor_j)) / std(factor_j)

// 加权求和
composite_score_i = Σ w_j × z_j,i

// 权重方案：等权（避免回测过拟合）或 条件IC加权
w_j ∝ |IC_j| （用训练集IC，非全样本）
```

**不推荐的聚合方式**：
- 排序后取rank再加权 → 在不同市场环境下rank的区分度变化很大
- 动态权重（每月按近期IC调整）→ 过拟合风险极高，样本外通常不如等权
