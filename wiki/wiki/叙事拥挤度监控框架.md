## 一、定位：串行架构的中间层

叙事拥挤度在整体regime框架中的位置：

```
月频 Regime 层（方向仓位）
  双轴四象限 → 基准仓位
  三重 Override（PB≥95% | PE≥95% | Gini≥0.68）→ 仓位=0
         ↓
周频 叙事拥挤度（本页面）          ← 新增
  narrative_multiplier [0.6-1.0]
         ↓
日频 Crash Alarm 层（实时风控）
  8个 alarm 指标 → 0/0.25/0.5/1 开关
         ↓
最终仓位 = 基准仓位 × override × 叙事乘数 × alarm开关
```

**为什么串行不是并行**：三重override处理尾部事件（"绝对不应该持仓"），叙事拥挤度处理非尾部但值得谨慎的时刻（"共识太紧，仓位轻一点"）。并行加权会稀释override的强制力。

**为什么是周频**：叙事凝聚/瓦解是2-6周尺度。月频太慢（错过预警窗口），日频噪音太大（每天看Gini波动无意义）。

## 二、第一性原理：叙事拥挤为什么预示风险

叙事拥挤 → 所有相信这个故事的人都已经买了 → 边际买家枯竭 → 任何风吹草动都会触发反转，且反转幅度远超正常波动。

这不是"市场情绪"的模糊判断。叙事拥挤有三个可被数据验证的特征：

1. **资金在往少数方向集中**（成交额HHI上升）
2. **定价被叙事绑架而非基本面**（板块内个股离散度下降）
3. **估值已被叙事推到高位**（PE分位数偏高但未到95%极端值）

- **当前（2026.6）Gini=0.736**，已突破0.68空仓线→与三重override的结论一致

## 三、周频叙事拥挤度指标

### 3.1 五个子指标

| 指标 | 频率 | 权重 | 逻辑 | 数据源 |
|------|------|------|------|--------|
| 成交额HHI（行业维度） | 周频 | 35% | 资金是否在往少数板块集中 | Tushare `sw_daily` 31个申万行业日成交额→周均→HHI |
| PE分位数（5年） | 周频 | 25% | 估值是否被叙事推到偏高位置（但未到95%极端值） | 现有 `panel_market_sentiment_monthly` |
| Gini指数 | 周频 | 25% | 全市场成交额分布不均度（0.50-0.68区间） | 现有面板 |
| ret_3m偏离度 | 周频 | 15% | 收益是否过度延伸（涨太多或跌太多都降低乘数） | 现有面板 |

**HHI计算公式**：
```
HHI_sector = Σ (sector_i_amount / total_market_amount)²
# 31个申万一级行业，周频取5日均值后计算
# 然后标到0-1：rolling_5y_percentile(HHI_weekly)
```

**与现有指标的关系**：
- `amount_gini` 衡量的是个股成交额不均度，HHI衡量的是行业/板块资金集中度——信息源独立
- Gini高但HHI低 = 同板块内几只权重股吸走成交（如茅台一只票吸走消费板块50%成交）
- Gini高且HHI高 = 少数板块的少数股票吸走全市场成交（最危险的状态）

### 3.2 不纳入的指标及理由

| 指标 | 排除理由 |
|------|---------|
| 板块内个股离散度 | 与 `cross_sectional_vol` 高度重叠，而CS vol已被排除（"对ETF轮动无帮助"） |
| 分析师评级分歧度 | A股卖方评级几乎全是买入/增持，分歧度信噪比太低 |
| 新发基金主题集中度 | 数据获取不稳定，改用手工跟踪→主观调阈值 |

## 四、crowding_score 合成与乘数映射

### 合成公式

```
crowding_score = HHI_zscore × 0.35
               + PE_pct_zscore × 0.25
               + Gini_zscore × 0.25
               + |ret_3m_zscore| × 0.15

# 每个子项先算rolling 5y分位数（0-1），然后合成
```

### 乘数映射（阶梯式，5档）

```
crowding_score < 0.30  → narrative_multiplier = 1.0   # 正常
[0.30, 0.50)          → narrative_multiplier = 0.9   # 微紧
[0.50, 0.70)          → narrative_multiplier = 0.8   # 偏紧
[0.70, 0.85)          → narrative_multiplier = 0.7   # 拥挤
≥ 0.85                → narrative_multiplier = 0.6   # 极度拥挤
```

阶梯式而非连续线性——避免过度拟合。5档够用。

### 与三重override的分工（避免重复计数）

| 区间 | PB/PE分位 | Gini | 谁管 |
|------|----------|------|------|
| 正常 | <75% | <0.50 | 都不管，乘数=1.0 |
| 偏贵 | 75%-90% | 0.50-0.65 | **叙事乘数**管（连续打折） |
| 极端 | ≥95% | ≥0.68 | **三重override**管（一票空仓） |

**关键设计**：同一指标在两种用法中没有重叠区间。override管尾部，乘数管正常区间。

## 五、日频Alarm层新增：北向-两融背离度

在现有7个日频crash alarm基础上，新增第8个：

```
alarm_north_margin_divergence:
  direction_north = sign(north_money_5d_sum)
  direction_margin = sign(margin_balance_5d_change)
  divergence = (direction_north != direction_margin)
  连续3个交易日 divergence → alarm触发 → 仓位上限50%
```

**逻辑**：北向（外资/机构）和两融（散户/杠杆资金）是两个信息源完全独立的资金群体。方向一致=叙事有合力，方向背离=共识破裂。

## 六、季频手工验证层

每季度公募季报披露后（1/4/7/10月下旬），手动跟踪两个数：

1. **Top 10重仓股集中度**：前10大重仓股占全部股票仓位的比例。>60%=拥挤
2. **主题押注比例**：某板块持仓超benchmark 1.5倍的基金数量占比。>40%=拥挤

这两个数**不直接参与crowding_score计算**——用于主观校准周频阈值。例如Top 10集中度突破60%且某主题基金发行占比>50%→手动把crowding_score警戒线从0.50调到0.40。

## 七、验证标准（复用风险因子物理评价体系）

叙事拥挤度的评估复用[[ETF策略风险指标与做空工具]]中的RAS/CCR/FAR框架：

```
RAS_bear: 拥挤度触发减仓后，在后续熊市中少亏了多少
Profit_Sacrifice: 拥挤度触发减仓后，在后续牛市中少赚了多少
CCR: 历史上每次大回撤（>5%）前，拥挤度是否提前预警
FAR: 拥挤度触发后，有多少次是假警报
```

目标区间：CCR 60-70%、FAR 30-40%。和日频alarm的评价标准一致。

## 九、Showcase关键结论

基于 [narrative_showcase.html](../narrative_showcase.html) 的2010-2026全周期数据：

1. **PE≥95%或Gini≥0.68在历史上几乎每次都出现在重大回调前**（2015.6、2021.2、2026.6）
2. **无叙事时期（2022、2023H2-2024Q3）指数横盘，拥挤度低≠安全，只是另一种难赚钱的regime**
3. **叙事拥挤度作为周频乘数，最大价值不是"预测顶部"（那是override的工作），而是"在偏贵但不到极端时逐步收力"**
4. **当前（2026.6）Gini=0.736已突破空仓线，与月频override结论一致**——这不是重复计数，是多层框架的交叉验证
