## 一、设计原则

1. **Sanity Check 优先**：任何 regime 定义必须与 A 股历史常识一致。锚点区间：
   - 2014H2-2015H1：全面牛（趋势↑ + 宽度↑）
   - 2016-2017：结构牛 / 蓝筹白马（趋势↑ + 宽度↓）
   - 2018：单边熊（趋势↓ + 宽度↓）
   - 2019-2020：结构牛（趋势↑ + 宽度部分↓）
   - 2022-2024Q3：震荡阴跌（趋势↓ + 宽度↓）
   - 2024Q4-2026H1：修复牛（趋势↑ + 宽度↑ → 后期宽度回落）
2. **先验约束搜索空间**：每个指标加入前必须有经济逻辑，参数是校准不是挖掘。PB 95%+ → 空仓有 60+ 年 Fama-French 先验支撑；Gini 0.68 有网络理论先验。
3. **月频管方向，日频管风控**：两套独立系统，不互相替代。

## 二、总体架构

```
月频 Regime 层（方向仓位）
=========================
方向轴 (ret3m, HS300)  ×  宽度轴 (pct_above_ma250, 全A)
          ↓                          ↓
    各自滞回二值化            各自滞回二值化
          ↓                          ↓
       四象限 regime ─────────────────────────┐
                                              │
三重 Override（空仓）                          │
  PB_pct_5y ≥ 0.95 OR PE_pct_5y ≥ 0.95        │
  OR amount_gini ≥ 0.68  → position = 0        │
                                              │
底仓 Override                                  ├──→ 月频仓位
  PB_pct_5y < 0.05  → position = max(pos, 0.5) │
                                              │
独立验证：amount_gini 佐证宽度真假              │
          spread(ZZ1000-HS300) 验证结构牛方向   │

=========================

          ↓
    独立 crash insurance 评估（不进月频回测）
```

## 三、轴 1：趋势方向 — HS300 idx_ret_3m + 滞回

| 参数 | 值 | 说明 |
|------|-----|------|
| 标的 | 沪深300 | 16-17 结构牛中 HS300 ret3m 83% 为正，全A 仅 58%；比上证50更宽 |
| upper | **0.07** | 涨 7% 确认趋势向上 |
| lower | **-0.03** | 跌 3% 确认趋势转弱（不对称：入难出易，符合 A 股牛短熊长） |
| switches | ~21 | 平均 8.5 月/段 |
| Bull% | ~50.3% | 一半时间在牛 |

**为什么 ret3m**：ret1m 自相关 0.07~0.10（随机），ret12m 自相关 0.91（太钝）。ret3m 0.65~0.68 恰好。

**参数来源**：网格搜索 U∈[0.01,0.12], L∈[-0.08,0.02]，7 锚点区间 ret3m 方向匹配率 + 熊市误判惩罚，一票否决 switches>30 或 bull%∉[20%,75%]。时间分割验证（训练 2010-2018，测试 2019-2026）：U=0.06 和 0.07 在训练集并列第一，L=-0.03 完全一致。

**⚠️ 每 12 个月重跑时间分割验证。**

## 四、轴 2：参与宽度 — 全A pct_above_ma250 + 滞回

| 参数 | 值 | 说明 |
|------|-----|------|
| 标的 | 全A | 宽度必须用全体股票 |
| upper | **0.70** | 70% 股票站上年线 = 高参与 |
| lower | **0.38** | 38% 跌破 = 宽度退潮 |
| switches | ~6 | 平均 30 月/段 |
| Bull% | ~20.7% | A 股大部分时间宽度低 |
| MA 平滑 | 不需要 | 截面值天然平滑 |

**为什么 ma250 不是 ma60**：ma60 对短期噪音敏感。ma250 站上 = 中长期趋势健康。**为什么全A 不是 HS300**：宽度是全体参与率，不是大票的健康度。

**参数来源**：固定 ret3m=0.07/-0.03，扫 pa250 U∈[0.52,0.70], L∈[0.30,0.40]。Top-5 全部指向 U=0.70。训练集最优 U=0.60（2010-2018 经历过 2014-15 疯牛不需要 U=0.70），全量收敛到 U=0.70（更保守）。

## 五、四象限

| | 趋势 Bull (ret3m ↑) | 趋势 Bear (ret3m ↓) |
|---|---|---|
| **宽度 Bull (pa250 ↑)** | 🟢 **全面牛** — 满仓、动量优先 | ⚪ **噪音过渡** — 保持上一期状态 |
| **宽度 Bear (pa250 ↓)** | 🟡 **结构牛** — 偏大票/指数 | 🔴 **熊市** — 空仓 |

噪音象限（趋势↓+宽度↑）约 15% 的月份，主要出现于 regime 切换过渡期。**不触发状态切换**——等价于自带一层滞回。

**回测验证**：全面牛月均 +2.2%（HS300/全A 一致），结构牛 HS300 月均 -0.07%（全A -0.23%）。结构牛买全A 累计亏 18.7%——四象限区分是真金白银。

## 六、三重 Override（一票空仓）

### 逻辑

```
if PB_pct_5y ≥ 0.95 OR PE_pct_5y ≥ 0.95 OR amount_gini ≥ 0.68:
    position = 0       # 不管 regime 说什么
elif PB_pct_5y < 0.05:
    position = max(position, 0.5)  # 至少半仓
```

**三个条件各司其职**：PB 极贵（资产价格不可持续）、PE 极贵（盈利预期脱离现实）、Gini 极高（资金集中→市场结构脆弱）。OR 逻辑——对个人投资者宁可误杀。不需要滞回——override 是纯阈值，PB 从 0.95 跌到 0.94 就解除。

### 数据验证

- PB_pct_5y 95-100%：fwd_12M = **-22.0%** (60月窗口, n=6)。36月窗口也是负
- amount_gini ≥ 0.68 (≈历史 92分位)：fwd_3M = **-3.6%**，fwd_6M = **-10.7%**。镜像：Gini < 10% 的 fwd_6M = +14.1%
- PE_pct_5y 95-100%：fwd_12M = -25.9%（n=5，fix-component 方法）

**回测效果（HS300）**：Regime only +80.9% → +Triple override **+245.8%**（年化 8.7%），最大回撤 -45.1% → **-12.4%**。

**⚠️ 这不是策略回测**——三重 override 在 2026 年 4 月把仓位打到 0，提前 3 个月回避了 2026 年 7 月的 crash。

| 指标 | 角色 | 关键发现 |
|------|------|---------|
| **amount_gini** | 宽度真伪验证器 | 高 Gini + 高宽度 = 伪宽（资金抱团少数票）。24Q4 Gini=0.635，比 2018 熊更高 |
| **spread(ZZ1000-HS300)** | 结构牛方向确认 | 16-17 结构牛 spread=-5.5%（大票碾压），24Q4 spread=+3.3%（小票碾压） |
| **avg_correlation** | 踩踏事后来看 | 月频滞后（飙升月=已崩盘月），保留日频做即时 alarm |
| **profit_ratio** | "局面在变"警报 | >0.80 出现在转折点附近，不区分顶还是底 |

### 被排除的指标

| 指标 | 排除理由 |
|------|---------|
| cross_sectional_vol | 衡量"选股能不能赚钱"，对 ETF 轮动无帮助 |
| turnover_rate_median（月频）| 涨跌都放量，月频不区分 |
| MA60/MA250 日频裸算 | 无滞回 = 裸奔，信号闪烁 |
| BBand（月频 2σ）| 9 次触发 7 次踏空——牛市持续贴着上轨走。且已被 PB/PE/Gini 覆盖 |
| MACD/RSI | ret3m 的信息子集（MACD≈MA 差，RSI≈涨幅映射） |
| downside_vol_ratio | 反向指标：高恐惧 = 该贪婪，不是该跑 |
| 技术分析（缠论/主升浪/神奇九转） | 不可证伪（缠论）、事后标签（主升浪）、假阳性高（九转） |

## 八、滞回（Hysteresis）精确定义

```
状态 s_t ∈ {0, 1}，初始 s_0 = 0

if s_{t-1} == 1 AND x_t < L:   s_t = 0    # 确认退出
elif s_{t-1} == 0 AND x_t > U: s_t = 1    # 确认进入
else:                           s_t = s_{t-1}
```

- **upper**：0→1 确认线。高 = 难进
- **lower**：1→0 确认线。高（接近 0）= 易退。低（更负）= 难退
- **switches** = Σ|s_t - s_{t-1}|：总切换次数
- 非对称设计（ret3m 入 7%/出 -3%）有直觉基础：A 股牛短熊长，进场需更严

## 九、日频 Crash Alarm 层

### 设计哲学
月频 regime 管方向仓位（每月一个决策窗口），日频 alarm 管实时风控（盘中触发 → 即时行动）。两套系统独立运行、独立评估，不互相嵌入回测。

### 日频 Alarm 候选池（7 个）

| 维度 | 指标 | 逻辑 | 月频为什么做不到 |
|------|------|------|------------------|
| 波 | **avg_correlation 日频** | 日内同涨同跌飙升 = 踩踏 | 月频均值抹平 spike |
| 波 | **vol-of-vol** (idx_vol_5d 的 20d std) | 波动率的波动率飙升 = regime 切换征兆 | 月频看不出来 |
| 价 | **连续 N 日创新低占比** | N 日内创 XX 日新低股票 > 50% = 恐慌广度 | 月频已消化 |
| 价 | **涨跌停比异常** | 涨停/跌停从 >3 骤降到 <0.5 | 月频平均值无意义 |
| 量 | **量价背离日数** | 连续 N 日量增价跌 → 分配信号 | 月频涨跌都放量，辨不了 |
| 量 | **尾盘量占比** | 尾盘 30 分钟成交 > 全天 40% = 恐慌出逃 | 日频独有 |
| 资金 | **北向单日异常流出** | 单日流出 > 历史 99% 分位 | 月频净流入抹平日内 |

**实施**：先拉 15 年日频分布，标出 6 次 crash（2015-06/2016-01/2018-02/2020-03/2024-01/2026-07）前 1-5 天是否触发极端值。≥4 次提前预警 = 可用。

### 月频指标 → 日频可行性总表

**✅ 日频增量明确（17 列）**：`profit_ratio`, `up_down_ratio`, `pct_above_ma60/250`, `limit_up_count`, `idx_amount`, `turnover_rate_median`, `amount_gini`, `idx_volatility_20/60`, `avg_correlation`, `cross_sectional_vol`, `north_money`, `margin_balance`, `net_inflow_ratio`, `inflow_breadth`, `institutional_pct`

**⚠️ 日频可算但噪声大/变化微小（13 列）**：估值类日均变化在小数点后第二位（虚假精度）；分位类（5y窗口日频）过密；稳定性类需 rolling 窗口再设计

**❌ 日频无意义（6 列）**：`ma60/250`（裸金叉无滞回=裸奔）、`idx_ret_1m/3m/12m`（日频 rolling ret ≠ 月频日历对齐+滞回信号、概念断裂）

**关键**：日频截面类（profit_ratio, pct_above, amount_gini, avg_correlation）可以直接平移，几乎无转换成本。日频趋势类（ret3m 月频 → rolling N 日 = 不同的东西）不值得平行维护，月频方向轴只存在于月频。

## 十三、开发 Spec（2026-07-20，合并所有 TODO）

> 池子原则：Phase 1-3 只用宽基 ETF（沪深300/中证500/创业板指/科创50/中证1000/上证50，6-8只）。原因：量比的 regime 翻转逻辑是股票市场特有的，黄金/商品/债券的放量驱动机制完全不同，混入会污染条件 IC。行业 ETF 波动特征差异大（消费低 ADX、券商高 ADX），应作为 Phase 4 用行业自身分布独立校准滞回阈值。

---

### Phase 1：条件 IC 验证（最优先，其他 Phase 依赖此步）

**目标**：用宽基 ETF 月频数据验证量比 regime 条件 IC 的四格点符号方向。

- [ ] **1.1 宽基 ETF 面板搭建**
  - ETF 清单：沪深300(510300)、中证500(510500)、创业板指(159915)、科创50(588000)、中证1000(512100)、上证50(510050)
  - 字段：日期、收盘价、成交额、MA50、StdDev_50、ATR_20
  - 时间范围：每只 ETF 上市日 ~ 2026-06
  - 输出：`etf_wide_panel_monthly.csv`

- [ ] **1.2 ADX 滞回计算（每只 ETF 独立）**
  - 每只 ETF 计算月频 ADX（14 日 rolling，取月末值）
  - 滞回二值化：upper=22, lower=18（初始值，Phase 5 再扫参）
  - 标记突破日：`trend_state` 从 False 翻 True 的月份
  - 输出列：`adx_raw`, `trend_state`, `is_breakout`

- [ ] **1.3 Z-score 计算**
  - `Z_score = (Close - MA_50) / StdDev_50`，月频
  - 分档：Z < 1.5（趋势中途） / Z ≥ 1.5（高位延伸）
  - 注意：每只 ETF 自身滚动计算，不跨 ETF 统一阈值

- [ ] **1.4 量比计算**
  - `volume_ratio = V_5d_avg / V_60d_avg`（月频用该月最后5日均量/60日均量）
  - forward return = 下月收益率

- [ ] **1.5 条件 IC 四格点验证**
  - 格点 1（突破日）：`is_breakout=True` 的子样本 → `corr(volume_ratio, fwd_ret)` → **预期 IC > 0**
  - 格点 2（趋势中途）：`trend_state=True, Z<1.5` → `corr(vr, fwd_ret)` → **预期 IC < 0**
  - 格点 3（高位延伸）：`trend_state=True, Z≥1.5` → `corr(vr, fwd_ret)` → **预期 IC < 0 且绝对值 > 格点2**
  - 格点 4（盘整）：`trend_state=False` → `corr(vr, fwd_ret)` → **预期 ≈ 0**
  - 额外验证：全样本 `corr(vr, fwd_ret)` → **预期 ≈ 0**（正负抵消）
  - ⚠️ 如果符号方向和预期不一致 → 暂停，不要继续往下走。回来看是 ADX 参数问题还是量比逻辑在 A 股宽基上不成立。

- [ ] **1.6 WAP 偏离条件覆盖测试**
  - 计算 `wap_dist = (Close - WAP_60) / ATR_20`
  - 按 `|wap_dist|` 分桶：<1.0 / [1.0, 2.0) / ≥2.0
  - 每桶内看 fwd_ret 分布（均值、标准差、右尾概率）
  - 目标：≥2.0 桶的右尾是否明显更肥（验证 WAP 偏离大时降权能否避开暴跌）

### Phase 2：entry_quality 调制器跑通

- [ ] **2.1 wap_penalty 实现**
  - 按 Section 6.1 简化版：`|wap_dist|<1→1.0, [1,2)→0.8, ≥2→0.5`
  - 可选：跑增强版八-C2 做对比（加 Z-score 交互），看边际增益

- [ ] **2.2 vol_factor 实现**
  - 按 Section 6.3 调制矩阵，四格点各一个 vol_factor 值
  - 映射：突破放量 1.5 / 中途放量 0.8 / 高位放量 0.5 / 正常 1.0

- [ ] **2.3 entry_quality 合成 + 动量串联**
  - `entry_quality = clamp(wap_penalty × vol_factor, 0.3, 1.5)`
  - 动量排名（3 个月收益）→ top 3 → `final_weight = rank_weight × entry_quality`
  - 基准对比：纯动量等权 × 动量+entry_quality —— 看回撤改善幅度

- [ ] **2.4 RAS_bear 评估**
  - 把回测按沪深300月收益正负切分
  - 熊市月：动量+EQ 组合收益 - 纯动量组合收益 → RAS_bear
  - 牛市月：动量+EQ 组合收益 - 纯动量组合收益 → Profit_Sacrifice
  - Net_Benefit = RAS_bear × 熊市占比 - Profit_Sacrifice × 牛市占比
  - > 0 → entry_quality 有效，< 0 → 退回去检查参数

## 十四、择时指标探查 Phase（2026-07-21，新增）

> 来源：股债性价比 + 恐慌贪婪指数的播客讨论。结论：不在现有框架外另起炉灶，而是补信息源 + 嵌入已有架构。

### 14.1 股债性价比 — 嵌入为 Override 或 Alarm

**核心问题**：E/P（盈利收益率）和10Y国债收益率有结构性漂移项——10Y从4.5%→1.7%的单边下行（约280bp/15年）远大于任何一年的周期性波动（±50bp）。spread用固定阈值会被结构趋势欺骗。

**修正方案（三选一，探查后定）**：

| 方案 | 做法 | 嵌入位置 | 优点 | 缺点 |
|------|------|---------|------|------|
| A. 去趋势化 | spread减36或60个月滚动均值，残差>1σ触发 | 新增override条件 | 简单，一个参数 | 窗口长度=超参数，不同窗口结论不同 |
| B. 拆开用 | E/P分位独立于10Y分位，两维交叉判断 | 新增双条件override | 无超参数，最诚实 | 两个独立判断的组合逻辑需要定 |
| C. 只做极端值 | spread在历史95分位以上→超配权益10-20%，其他时候不管 | 仓位粗调（类似底仓override方向） | 最省事 | 放弃大部分信息 |

**推荐探查路径**：
- [ ] **14.1.1** 拉沪深300 E/P（1/PE_ttm）和10Y国债收益率 2010-2026 月频序列
- [ ] **14.1.2** 计算原始 spread = E/P - 10Y，画时序图 → 肉眼确认结构漂移的量级
- [ ] **14.1.3** 方案A探查：去趋势spread（36月/60月两个窗口）的 fwd_3m/6m/12m return 条件分布
- [ ] **14.1.4** 方案B探查：E/P 5y分位 和 10Y 5y分位的 2×2 四象限，每象限 fwd return
- [ ] **14.1.5** 方案C探查：spread > 历史95分位 之后12个月的 max drawdown 和 return
- [ ] **14.1.6** 和现有 override（PB/PE/Gini）的共线时间点排查——如果触发时段重叠≥80%，无独立信息量，不加入
- [ ] **14.1.7** 定嵌入位置：override（一票否决仓位）还是仓位粗调（偏配±20%）——取决于信号强度

**证伪条件**：三种修正方案中至少一种的 fwd return 条件分布与无条件分布有统计显著差异 → 可用。否则股债性价比在A股不成立，砍掉。

**名义/实际变量差异（需理解，不是实现细节）**：
- 通胀冲击时，10Y立刻反应（利率跳升），E/P滞后3-4季度反应（盈利最终随通胀上升）。短期spread波动有一半是反应速度差，不是相对价值变化。
- 这意味着spread在宏观波动大的年份（2022）噪声特别大。去趋势化方案A能部分缓解，但不能根除。

### 14.2 恐慌贪婪指数 — 不建独立指数，补三个 Alarm

**结论**：现有 crash_alarm_daily 的验证逻辑（6次crash事件回溯→forward return条件分布）已成熟。恐慌贪婪的增量价值不在换包装（连续分数 vs 布尔alarm），在**补三块未覆盖的独立信息源**。

**新增三个 Alarm**：

| # | Alarm | 数据源 | 逻辑 | 现有覆盖？ |
|---|-------|--------|------|----------|
| 9 | `alarm_futures_contango` | `fut_daily`（主力连续结算价）+ `index_daily`（现货收盘） | 贴水超过过去250日95%分位 = 对冲需求涌入 = 聪明钱恐慌。A股期权市场深度不够，期货升贴水比Put/Call比更有效 | ❌ 完全缺失 |
| 10 | `alarm_margin_buy_pct` | `margin_detail`（buy_amount）+ 全市场成交额 | 融资买入/总成交 > 过去250日95%分位 = 散户杠杆贪婪顶。顶部信号比底部信号更可靠 | ❌ 有margin_balance但无买入占比 |
| 11 | `alarm_credit_spread` | `bond_blk`（曲线估值，AA-AAA利差） | 利差走阔 > 过去250日95%分位 = 信用风险定价上升。债市领先股市，信用利差是宏观恐慌的领先指标 | ❌ 完全缺失 |

**和现有 8 个 alarm 的信息源独立性**：

| 现有 alarm | 信息维度 | 新增 alarm | 重叠？ |
|------------|---------|-----------|--------|
| alarm_avg_corr_spike | 相关性/踩踏 | 升贴水 | 无重叠，期货是机构对冲行为 |
| alarm_vol_of_vol | 波动率突增 | 升贴水 | 无重叠 |
| alarm_new_low_pct | 恐慌广度 | 信用利差 | 无重叠，信用市场独立 |
| alarm_limit_ratio | 涨跌停情绪 | 融资买入 | 部分重叠（都是散户情绪），但融资是杠杆维度 |
| alarm_vol_price_divergence | 量价背离 | 融资买入 | 无重叠 |
| alarm_tail_volume | 尾盘量异常 | — | — |
| alarm_north_outflow | 北向恐慌 | 升贴水 | 部分重叠（都是聪明钱），但机构类型不同（外资 vs 内资对冲） |
| alarm_adx_low | 趋势缺位 | — | — |

三个新 alarm 最多和现有 `alarm_north_outflow` / `alarm_limit_ratio` 有轻度信息重叠，不算冗余。

**探查步骤**：
- [ ] **14.2.1 数据源可达性验证**：检查 `fut_daily` / `margin_detail` / `bond_blk` 的 tushare 积分权限和数据覆盖范围
- [ ] **14.2.2** 期货升贴水：计算 IF/IC 主力连续合约的 basis = (settle - index_close) / index_close，拉 2015-2026 日频分布
- [ ] **14.2.3** 融资买入占比：`buy_amount / 全市场日成交额`，拉 2015-2026 日频，确认滚动分位阈值
- [ ] **14.2.4** 信用利差：AA - AAA 收益率差值，核实 bond_blk 数据质量。如果 tushare 质量不可靠，标注为中债登手动补
- [ ] **14.2.5** 三指标各自极端值（>95%分位 / <5%分位）的 fwd_1w/1m/3m/6m return 条件分布
- [ ] **14.2.6** 6 次 crash 事件回溯（2015-06/2016-01/2018-02/2020-03/2024-01/2026-07）：新增 alarm 是否在 crash 前 1-5 天触发？
- [ ] **14.2.7** 和现有 alarm 的共发率：如果新 alarm 触发时 >70% 的时间已有其他 alarm 触发 → 无独立信息量，不加入
- [ ] **14.2.8** 定 alarm 逻辑：每个新 alarm 是独立触发（OR）还是需要和现有 alarm 组合触发（AND）

**证伪条件**：≥1 个新 alarm 在 ≥4/6 次 crash 事件前触发 → 加入 alarm 候选池。0 个通过 → 恐慌贪婪在 A 股暂时缺乏独立信号，标记为"待数据源完善后复探"。

**数据优先级**：升贴水 > 融资买入 > 信用利差。理由：升贴水信息维度最独立（期货市场 = 对冲需求），融资买入在散户占比下降趋势中信号可能衰减，信用利差受 tushare 数据质量约束最大。
